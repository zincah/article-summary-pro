## 1. 서비스 소개

### 1.1 ArticleSummary Pro란?

ArticleSummary Pro는 **AI 기반 기사 요약 서비스**입니다.
바쁜 현대인들을 위해 긴 기사의 핵심 내용을 몇 초 만에 파악할 수 있도록 도와줍니다.

**주요 특징:**
- 🚀 **빠른 요약:** 평균 2-3초 내 요약 완료
- 🎯 **정확한 내용:** Claude AI의 최신 모델 활용
- 📱 **모바일 친화:** 스마트폰, 태블릿, PC 모두 지원
- 💾 **히스토리:** 요약한 기사 자동 저장
- 🎨 **커스터마이징:** 요약 길이와 톤 조절 가능
- 🌙 **다크모드:** 야간 모드 지원
- 🆓 **무료:** 추가 비용 없음

### 1.2 언제 사용하면 좋을까?

| 상황 | 사용 예시 |
|------|---------|
| **출근길** | 뉴스레터의 주요 기사를 빠르게 훑어보기 |
| **점심시간** | SNS에서 본 흥미로운 기사의 내용 확인 |
| **독서** | 책에서 인용된 기사를 빠르게 파악 |
| **학습** | 리포트나 과제를 위한 배경 자료 수집 |
| **업무** | 경쟁사/시장 분석 기사를 효율적으로 리뷰 |

## 3. 기능별 사용 방법

### 3.1 URL로 기사 요약하기 (가장 빠른 방법)

**1단계: URL 복사**
- 요약하고 싶은 기사의 주소를 복사합니다
- 예: https://news.naver.com/main/read.naver?oid=001&aid=0013905402

**2단계: URL 입력**
- ArticleSummary Pro 메인 화면 열기
- "📎 URL 입력" 탭에서 복사한 URL을 붙여넣기
- 또는 직접 주소를 입력

**3단계: 요약 생성**
- (선택) 길이 선택: 짧음 / 중간 / 길음
- (선택) 톤 선택: 중립 / 전문 / 쉬움
- "[요약 생성]" 버튼 클릭

**4단계: 결과 확인**
- 실시간으로 요약이 생성됩니다 (타이핑 효과)
- 완료 후 자동으로 히스토리에 저장됩니다
- 필요시 [복사], [즐겨찾기], [삭제] 가능

**팁:**
- 잘못된 URL을 입력하면 오류 메시지가 표시됩니다
- "중단" 버튼을 누르면 생성 중인 요약을 멈출 수 있습니다
- 로딩 중 페이지를 떠나도 안전합니다

### 3.2 텍스트로 요약하기 (로그인 필요 사이트용)

**상황:**
- 기사가 로그인 뒤에 있는 경우
- 뉴스레터나 이메일의 본문을 요약하고 싶을 때
- 자신이 작성한 텍스트를 정리하고 싶을 때

**1단계: 텍스트 입력**
- "📝 텍스트 직접 입력" 탭 선택
- 요약할 텍스트를 복사하여 붙여넣기
- 또는 직접 입력

**2단계: 글자 수 확인**
- 입력 필드 아래의 진행 표시기 확인
- 최대 5,000자까지 입력 가능
- 추천: 200자 ~ 3,000자 (가장 좋은 결과)

**3단계: 요약 생성**
- URL 방식과 동일 (길이/톤 선택 후 [요약 생성])

**4단계: 결과 확인**
- URL 방식과 동일

**팁:**
- 페이지 본문만 복사 (광고, 메뉴 제외)
- 이미지나 표는 텍스트로 변환 후 입력
- 너무 짧은 텍스트(<100자)는 요약이 제대로 안 될 수 있음

### 3.3 요약 길이 조절

**세 가지 길이 옵션:**

| 옵션 | 설명 | 사용 시기 | 예상 길이 |
|------|------|---------|---------|
| **짧음** | 한 문장 ~ 3문장 | 빠르게 개요만 파악하고 싶을 때 | 50-100자 |
| **중간** (기본값) | 3문장 ~ 5문장 | 일반적인 사용 | 150-300자 |
| **길음** | 5문장 ~ 10문장 | 자세한 내용이 필요할 때 | 400-600자 |

**선택 방법:**
1. 요약 생성 전: "길이" 옵션에서 선택 후 [요약 생성]
2. 요약 생성 후: "길이" 옵션 변경 후 [다시 생성]

### 3.4 요약 톤 조절

**세 가지 톤 옵션:**

| 옵션 | 설명 | 사용 시기 | 예시 |
|------|------|---------|------|
| **중립** | 객관적이고 감정 없이 | 뉴스나 팩트 중심 기사 | "A사는 신제품을 출시했다." |
| **전문** (기본값) | 업계 용어와 깊이 있게 | 경제, 기술, 법률 기사 | "A사는 고부가가치 솔루션을 시장에 투입했다." |
| **쉬움** | 초등학생도 이해할 수 있게 | 일반 대중 대상 설명 | "A사가 새로운 물건을 만들었어요." |

**선택 방법:**
1. 요약 생성 전: "톤" 옵션에서 선택 후 [요약 생성]
2. 요약 생성 후: "톤" 옵션 변경 후 [다시 생성]

### 3.5 복사 및 공유

**결과 복사하기:**
1. 요약 생성 완료 후 [📋 복사] 버튼 클릭
2. 클립보드에 복사됨 (브라우저에서 자동 감지)
3. 이메일, 메시지, 문서 등에 붙여넣기 가능

**원문 보기:**
- "📄 원문 (일부)" 섹션에서 [전체 보기] 클릭
- 스크래핑된 전체 본문 확인 가능
- 원문도 함께 복사 가능

### 3.6 히스토리 관리

**히스토리 화면 접근:**
1. 하단 네비게이션에서 "📋 히스토리" 탭 클릭
2. 생성한 모든 요약이 날짜별로 그룹화되어 표시됨

**히스토리 기능:**

| 기능 | 설명 | 방법 |
|------|------|------|
| **보기** | 과거 요약 다시 확인 | "[보기]" 버튼 클릭 |
| **즐겨찾기** | 중요한 요약 표시 | "[❤️]" 버튼 클릭 (채워짐) |
| **삭제** | 히스토리에서 제거 | "[✕]" 버튼 클릭 |
| **검색** | 특정 요약 찾기 | (향후 제공 예정) |

**데이터 저장:**
- 모든 히스토리는 브라우저의 로컬 스토리지에 저장
- 서버에 저장되지 않음 (프라이버시 보호)
- 브라우저 캐시 삭제 시 히스토리도 삭제됨

**히스토리 초기화:**
1. ⚙️ 설정 탭 클릭
2. "히스토리 초기화" 버튼 클릭
3. 확인 후 모든 히스토리 삭제

### 3.7 설정 및 개인화

**설정 화면 접근:**
1. 하단 네비게이션에서 "⚙️ 설정" 탭 클릭
2. 개인 선호도 설정 가능

**설정 항목:**

| 항목 | 옵션 | 설명 |
|------|------|------|
| **기본 길이** | 짧음/중간/길음 | 매번 선택하지 않아도 기본값 적용 |
| **기본 톤** | 중립/전문/쉬움 | 매번 선택하지 않아도 기본값 적용 |
| **다크모드** | ON/OFF | 화면 배경색 변경 (야간 모드) |
| **데이터 수집** | ON/OFF | 서비스 개선을 위한 익명 통계 수집 동의 |
| **히스토리 초기화** | 버튼 | 저장된 모든 요약 삭제 (되돌릴 수 없음) |

   - 이 방법으로 대부분의 사이트 요약 가능

2. **피드백 보내기:**
   - 설정 > 피드백 보내기
   - "지원되지 않는 사이트: [사이트명]" 보내주기
   - 인기도가 높으면 향후 지원 추가

## 5. 자주 묻는 질문 (FAQ)

### Q1: 정말 무료인가요?
**A:** 네, 완전히 무료입니다. 회원가입, 광고 시청, 결제 없이 누구나 이용 가능합니다.

### Q2: 내 데이터는 어디에 저장되나요?
**A:** 모든 히스토리와 설정은 당신의 브라우저에만 저장됩니다 (로컬 스토리지). 서버에 저장되지 않아 프라이버시가 완전히 보호됩니다.

### Q3: 히스토리가 다른 기기에서도 보이나요?
**A:** 아니오, 로컬 스토리지이므로 해당 브라우저에서만 보입니다. 다른 기기나 브라우저에서는 별도로 저장됩니다.

### Q4: 요약이 정확한가요?
**A:** Claude AI의 최신 모델(Claude Sonnet 4.6)을 사용하여 높은 정확도를 유지합니다. 다만 모든 AI와 마찬가지로 완벽하지는 않을 수 있습니다. 중요한 내용은 원문 확인을 권장합니다.

### Q5: 요약 시간은 얼마나 걸리나요?
**A:** 평균 2-3초입니다 (텍스트 요약). URL 요약은 웹사이트 응답 속도에 따라 3-5초 소요될 수 있습니다.

### Q6: 동시에 여러 기사를 요약할 수 있나요?
**A:** 한 번에 하나씩만 요약됩니다. 요약 중에 중단 버튼으로 멈출 수 있습니다.

### Q7: 대충 요약된 것 같아요. 어떻게 하나요?
**A:** 다음을 시도해보세요:
1. 다른 "톤"으로 다시 생성해보기 (중립/전문/쉬움)
2. 다른 "길이"로 다시 생성해보기 (짧음/중간/길음)
3. 여전히 마음에 안 들면 피드백 보내기

### Q8: 정치, 종교, 논란이 되는 주제도 요약되나요?
**A:** 네, 주제에 관계없이 객관적으로 요약합니다. 단, AI의 특성상 일부 편향이 있을 수 있으므로 중요한 결정은 여러 출처의 원문을 확인하세요.

### Q9: 외국 뉴스도 요약되나요?
**A:** 네, 영어, 중국어, 일본어 등 다양한 언어를 지원합니다. 현재 버전은 원문과 같은 언어로 요약됩니다.

### Q10: 요약 길이가 너무 짧거나 길어요.
**A:** 길이 옵션을 변경해보세요:
- "짧음" - 한두 문장 (빠르게 개요만)
- "중간" (기본) - 3-5문장 (일반적 사용)
- "길음" - 5-10문장 (자세한 내용)

### Q11: 로그인 없이도 사용할 수 있나요?
**A:** 네, 완전히 로그인 없이 사용 가능합니다. 향후 계정 기능은 선택사항이 될 예정입니다.

### Q12: 오프라인에서 사용할 수 있나요?
**A:** 아니오, 요약 생성에는 인터넷 연결이 필수입니다 (Claude API 호출). 히스토리는 오프라인에서 볼 수 있습니다.

### Q13: 스크린샷을 요약할 수 있나요?
**A:** 아니오, 현재는 텍스트와 URL만 지원합니다. 스크린샷의 텍스트를 복사하여 "텍스트 직접 입력"에서 사용하세요.

### Q14: 비디오 콘텐츠를 요약할 수 있나요?
**A:** 아니오, YouTube 영상 등의 자동 요약은 지원되지 않습니다. 영상의 스크립트를 텍스트로 복사하여 입력하면 요약 가능합니다.

### Q15: 요약 결과를 PDF로 저장할 수 있나요?
**A:** 현재는 직접 지원하지 않습니다. 브라우저의 인쇄 기능 (Ctrl+P / Cmd+P) > "PDF로 저장" 사용하거나, 텍스트를 복사하여 문서에 붙여넣으세요.

### Q16: 피드백이나 버그는 어디에 보고하나요?
**A:** 설정 화면의 "피드백 보내기" 버튼을 클릭하여 의견을 보내주세요. GitHub Issues에서도 신고 가능합니다.

## 6. 문제 해결

### 문제: "요약이 안 되고 로딩만 계속돼요"

**원인 및 해결:**
1. **인터넷 연결 확인**
   - 다른 웹사이트 방문 시도 (예: google.com)
   - WiFi 재연결 또는 모바일 데이터 확인

2. **API 서버 상태 확인**
   - 개발자 도구 (F12) > Network 탭에서 요청 확인
   - 요청이 완료되지 않으면 서버 대기 중

3. **브라우저 캐시 삭제**
   - 브라우저 설정 > 캐시 및 쿠키 삭제
   - 페이지 새로고침 (Ctrl+R 또는 Cmd+R)

4. **다른 브라우저 시도**
   - Chrome, Firefox, Safari 중 다른 것 시도
   - 특정 브라우저에서만 문제인지 확인

### 문제: "URL을 입력했는데 '접근할 수 없습니다'라는 오류"

**원인 및 해결:**
1. **유료 사이트 (Paywall)**
   - WSJ, Bloomberg, FT 등 유료 뉴스는 불가
   - 로그인 후 텍스트 직접 입력으로 대체

2. **로그인 필요 사이트**
   - Facebook, Instagram, LinkedIn 등
   - 텍스트 직접 입력으로 대체

3. **사이트 주소 오류**
   - URL을 다시 확인하세요
   - http:// 또는 https://로 시작하는지 확인

4. **지원하지 않는 사이트**
   - 설정 > 피드백 보내기로 신청

### 문제: "요약 결과가 형편없어요"

**해결 방법:**
1. **다른 길이 시도**
   - "길음" 옵션으로 더 자세한 요약 생성

2. **다른 톤 시도**
   - "중립"이나 "쉬움" 옵션으로 다시 생성

3. **원문 확인**
   - "📄 원문 (일부)"의 "[전체 보기]"로 스크래핑된 텍스트 확인
   - 불완전하면 텍스트 직접 입력으로 대체

4. **피드백 보내기**
   - 설정 > 피드백 보내기
   - 어떤 기사에서 문제인지 알려주면 개선 가능

### 문제: "히스토리가 사라졌어요"

**원인:**
- 브라우저 캐시/쿠키 삭제
- 브라우저 자체 캐시 삭제
- 브라우저의 저장소 관리에서 로컬스토리지 삭제

**복구 불가:** 로컬 스토리지이므로 삭제되면 복구 불가. 정기적인 백업 권장.

### 문제: "복사 버튼이 작동하지 않아요"

**해결:**
1. **보안 맥락 확인**
   - HTTPS 연결 확인 (주소창의 자물쇠 아이콘)
   - HTTP 사이트에서는 복사 기능 제한될 수 있음

2. **브라우저 권한 확인**
   - 주소창 > 자물쇠 > 사이트 설정
   - "클립보드" 권한을 "허용"으로 설정

3. **다른 방법**
   - 마우스로 텍스트 선택 > Ctrl+C (Cmd+C) 복사

### 문제: "다크모드가 적용 안 됨"

**해결:**
1. **설정 확인**
   - ⚙️ 설정 > "다크모드" ON으로 설정
   - 페이지 새로고침

2. **캐시 삭제**
   - 브라우저 캐시 삭제 후 재방문

3. **시스템 설정**
   - 브라우저가 시스템 다크모드 따르도록 설정되어 있는지 확인
