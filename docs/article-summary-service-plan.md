## 1. 서비스 기획

### 1.1 서비스 개요 및 목표

**서비스명:** ArticleSummary Pro

**비전:** 바쁜 현대인들이 시간 소모적인 긴 기사 읽기의 부담에서 벗어나 핵심 내용만 빠르게 파악할 수 있는 AI 요약 서비스

**핵심 목표:**
- 사용자가 5분 이내에 기사의 핵심 내용 파악 가능
- 높은 정확도의 요약(원문 의미 손실 최소화)
- 직관적이고 빠른 사용자 경험

---

### 1.2 타겟 사용자 정의

| 세그먼트 | 특성 | 주요 니즈 | 사용 시나리오 |
|---------|------|---------|-----------|
| **바쁜 직장인** (Primary) | 30-45세, 높은 정보 소비욕 | 빠른 정보 습득 | 출근길, 점심시간에 뉴스 스캔 |
| **학생/연구자** (Secondary) | 20-30세, 학습용 | 다량의 자료 빠른 파악 | 리포트 준비, 논문 배경 조사 |
| **저널리스트/에디터** | 언론사 종사자 | 기사 검수, 트렌드 추적 | 다량의 경쟁사 기사 모니터링 |
| **콘텐츠 크리에이터** | SNS/블로그 운영자 | 다양한 주제 빠른 습득 | 콘텐츠 영감 수집 |

**주요 가정:**
- PC/모바일 모두에서 활발한 사용
- 기사 출처: 온라인 뉴스, 블로그, 매체 기사
- 평균 세션 시간: 5-15분
- 월 활성 사용자: 최초 1만 명 → 6개월 후 10만 명

---

### 1.3 핵심 기능 정의

#### MVP (1개월) - Core Loop: 기사 입력 → 요약 생성 → 결과 확인

| 기능 | 설명 | 우선순위 | 복잡도 |
|------|------|---------|--------|
| URL 기반 기사 입력 | 뉴스 링크 붙여넣기 | P0 | Low |
| 텍스트 직접 입력 | 복사한 텍스트 직접 입력 | P0 | Low |
| AI 자동 요약 | Claude API 이용한 1단계 요약 | P0 | Medium |
| 요약 결과 표시 | 원문/요약 병렬 보기 | P1 | Low |
| 복사 기능 | 요약 결과 클립보드 복사 | P1 | Very Low |
| 요약 길이 조절 | 짧음/중간/길음 선택 | P1 | Low |

#### Phase 2 (2-3개월)
- 요약 이력 저장 (세션 기반 로컬 스토리지)
- 즐겨찾기 기능
- 요약 재생성 (다른 톤/스타일)
- 핵심 키워드 추출 및 표시
- 다국어 지원 (한영일중)

#### Phase 3 (4-6개월)
- 사용자 계정 및 요약 이력 동기화
- 주제별 요약 스타일 커스터마이징
- 추천 알고리즘
- 브라우저 확장프로그램
- API 공개 (B2B 제휴)

### 1.5 경쟁 서비스 분석

| 서비스 | 장점 | 약점 | 우리의 차별점 |
|--------|------|------|-----------|
| **ChatGPT** | 강력한 AI | 기사 요약 특화 X, 비쌈 | 기사 요약 특화, 간단한 UI |
| **Naver Clova** | 한국어 최적화, 빠름 | 폐쇄형 API | 투명한 API, 오픈 기여 |
| **Upword** | 뉴스 큐레이션 | 구독료, 복잡 | 단순하고 무료 |
| **Summari** | 다국어, 정확 | 느림, 제한적 | 빠른 응답, 높은 커스터마이징 |

**차별화 전략:**
1. 단순성: 한 번의 클릭으로 요약 완료
2. 속도: 평균 응답 시간 < 3초
3. 가격: 초기 무료
4. 투명성: 오픈 소스 기여, 빠른 피드백 반영
5. 커스터마이징: 요약 톤, 길이, 포맷 유연함

## 2. UI/UX 기획

### 2.1 사용자 여정

```
[방문] → [기사 입력] → [요약 생성] → [결과 확인] → [공유/저장]
  ↑                                                    ↓
  └─────────── [히스토리 확인] ← [마이페이지 접근] ←─┘
```

**각 단계별 예상 시간:**
- 방문 → 입력: 10초
- 입력 → 요약 생성: 3-5초
- 결과 확인: 30-60초
- 공유/저장: 5-10초
- **Total: 2-3분**

### 3.2 Claude API 활용 방안

#### API 호출 흐름

```
사용자 입력 (URL/텍스트)
    │
    ▼
입력 검증 (유효성, 길이)
    │
    ▼
본문 추출 (URL: fetch+Cheerio, 텍스트: 직접)
    │
    ▼
```

```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();

async function summarizeArticle(
  content: string,
  options: {
    length: "short" | "medium" | "long";
    tone: "neutral" | "professional" | "simple";
  }
) {
  const lengthMap = {
    short: "1-2문단, 50-100단어로",
    medium: "3-4문단, 150-250단어로",
    long: "5-7문단, 300-500단어로",
  };

  const toneMap = {
    neutral: "객관적이고 감정 없이",
    professional: "전문 용어를 유지하면서",
    simple: "일반인도 이해하기 쉽게",
  };

  const message = await client.messages.create({
    model: "claude-3-5-sonnet-20241022",
    max_tokens: 1024,
    system: `당신은 경험 많은 전문 저널리스트입니다. 
            다음 기사를 ${lengthMap[options.length]}, 
            ${toneMap[options.tone]} 요약해주세요.
            마크다운 형식으로 작성하세요.`,
    messages: [
      {
        role: "user",
        content: `다음 기사를 요약해주세요:\n\n${content}`,
      },
    ],
  });

  return message.content[0].type === "text" ? message.content[0].text : "";
}
```

## 4. 추가 고려사항

### 4.1 AI 모델의 한계 및 대처

| 한계 | 증상 | 대처 |
|------|------|------|
| **환각(Hallucination)** | 원문에 없는 내용 추가 | 원문/요약 병렬 표시, 피드백 수집 |
| **길이 제어** | 길이 기준 애매함 | 토큰 제한 + 사용자 피드백 학습 |
| **문맥 손실** | 긴 기사의 핵심 누락 | 청크 단위 처리 후 통합 |
| **언어 편향** | 비한국어 콘텐츠 부정확 | 입력 언어 감지, 필요시 번역 |
| **비용 증가** | 사용량 폭증 시 비용 급증 | 사용자당 제한 + 프리미엄 모델 |

### 4.2 규제 및 준수

- **저작권**: 기사 요약은 변형 저작물
  - 출처 명시 (원문 링크 제공)
  - 이용약관에 보호 조항 포함
  - 저작권 클레임 시 즉시 삭제 프로세스

- **개인정보보호법**:
  - 사용자 동의 없는 데이터 수집 금지
  - 로컬 스토리지 활용 (서버 저장 최소화)
  - 삭제 권리 보장

- **AI 윤리**:
  - 기사 왜곡 방지 (정치, 사회 이슈 주의)
  - 자동 콘텐츠 생성임을 명시
  - 신뢰할 수 없는 요약에 대한 면책
